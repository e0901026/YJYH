# V0.5 范围确认

> 阶段目的：在 Android 核心流程和真实扫码通过后，建设真实后端最小可用版本，为后续 Android 联调做准备。

## 1. 版本定位

V0.5 建议定位为：后端 MVP 与数据库落地版。

V0.5 先让服务端核心业务跑起来，Android 端仍保留 mock repository，不在本阶段强制切换真实 API。等后端接口稳定后，V0.6 再做 Android 真实 API 联调。

## 2. V0.5 目标

- 创建 Spring Boot 后端工程。
- 接入 PostgreSQL 和 Flyway 数据迁移。
- 实现认证、用户、设备、借还、邀请码、通知记录核心 API。
- 实现 App 行为事件 / 错误事件接收 API。
- 保证 API 字段能支撑当前 Android 页面，不要求 Android 立即切换。
- 建立后端自动化测试和 CI 质量门禁。
- 输出可供 Android V0.6 联调使用的接口说明。

## 3. 范围内

### 后端工程

- 新增后端服务目录。
- Spring Boot 单体服务。
- 分层建议：controller / service / repository / domain / dto。
- 统一错误响应结构。
- 基础鉴权过滤器或安全配置。

### 数据库

- PostgreSQL 为目标数据库。
- Flyway 管理建表迁移。
- 首批表：users、invite_codes、devices、loan_records、notifications。
- 暂不实现 owner_transfer_records，除非 V0.5 开发中发现当前页面必须依赖。

### 核心 API

认证：

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/refresh`

当前用户：

- `GET /api/me`

设备：

- `GET /api/devices`
- `GET /api/devices/{id}`
- `GET /api/devices/by-imei/{imei}`
- `POST /api/devices`

借还：

- `POST /api/loans/borrow-by-imei`
- `GET /api/loans/active`
- `POST /api/loans/{id}/return`
- `POST /api/loans/{id}/urge-return`

邀请码：

- `GET /api/invite-codes/my`
- `POST /api/invite-codes/apply`
- `GET /api/owner/invite-codes`
- `POST /api/owner/invite-codes`

Owner：

- `GET /api/owner/users`

通知：

- `GET /api/notifications`
- `POST /api/notifications/{id}/read`

### 测试

- 后端单元测试覆盖核心 service 规则。
- 接口测试覆盖认证、设备 IMEI 查询、借机、归还、邀请码、事件上报。
- 数据库迁移可在本地测试库跑通。
- CI 至少执行后端构建与测试。

## 4. 范围外

- Android 全量切换真实 API。
- 生产部署、域名、证书和正式环境运维。
- 真实推送服务。
- 后台 Web 管理端。
- 复杂审批流。
- 审计导出。
- 多租户 / 多组织隔离。
- 性能压测和高可用架构。

## 5. 开发准入

进入 V0.5 后端开发前必须满足：

- V0.4 Android 真实扫码已人工验收通过。
- 本文 V0.5 范围得到用户确认。
- 后端与数据库架构文档已作为实现依据。
- 若 API 字段需要改变 Android 页面展示，必须先回到 PRD 和原型确认。

## 6. 验收标准

- 后端工程能本地启动。
- Flyway 能创建核心表。
- 注册、登录、当前用户接口可用。
- 根据 IMEI 查询设备接口可用。
- 未建档 IMEI 可创建设备。
- 已建档 IMEI 可创建借机记录并更新当前持有人。
- 一键还可结束 active 借用记录。
- 催还机可创建通知记录，不改变归还状态。
- 邀请码申请和 Owner 生成邀请码规则可用。
- App 事件上报接口可用，能记录用户行为和错误事件。
- 后端自动化测试通过。
- CI 可执行后端构建和测试。

## 7. 建议开发顺序

1. 创建后端工程脚手架和本地运行说明。
2. 增加数据库连接、Flyway 和首批迁移。
3. 实现用户、认证和 token 基础能力。
4. 实现设备和 IMEI 查询。
5. 实现借机、归还、催还通知记录。
6. 实现邀请码和 Owner 用户列表。
7. 补齐接口测试和 CI。
8. 输出 V0.6 Android 联调准备清单。

## 8. 待用户确认

- V0.5 是否按“后端 MVP 优先，Android 暂不切真实 API”推进。
- V0.5 是否先覆盖以上核心 API，不做后台 Web 管理端。
- V0.5 是否允许新增后端工程目录和后端 CI。
