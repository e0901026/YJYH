# 手机借机管理 App — 系统设计规范

> 目的：让所有页面保持同一套移动端低保真产品风格，避免单页风格漂移。

## 1. 设计目标

- 企业内部工具，风格应克制、清晰、可扫描。
- 以任务效率优先，不做营销页、装饰性大图、渐变背景或过度视觉表现。
- 首页可以突出核心指标，但必须和其他页面共享同一套视觉语言。

## 2. 视觉基线

| 项目 | 规范 |
|------|------|
| 页面尺寸 | 移动端竖屏，当前原型使用 390 × 844 |
| 页面背景 | 浅灰：`#F6F8FA` |
| 卡片背景 | 白色：`#FFFFFF` |
| 主色 | 蓝色：`#1677FF` |
| 主色浅底 | `#EAF3FF` |
| 正文 | `#17212B` |
| 次级文字 | `#697586` |
| 描边 | `#D9E1E8` |
| 成功 | `#0E9F6E` |
| 警示 | `#F59E0B` |
| 错误 | `#D92D20` |

## 3. 布局规则

- 页面外边距：左右 18px 左右，页面内模块间距 12px 到 14px。
- 卡片统一白底、1px 浅色描边、8px 圆角。
- 页面标题使用 26px、700 字重。
- 流程页返回必须使用标题栏左侧返回图标，遵循 Android 常规返回逻辑；不得把返回作为右上角文字链接。
- 标题栏右侧仅放当前页面的补充动作，例如设备列表的添加入口使用 `+` 图标。
- 设备入口和设备 Tab 必须使用能明确表达手机/设备的图标，不使用抽象方框或难以识别的符号。
- 卡片标题使用 15px 到 16px、700 字重。
- 辅助说明使用 12px 到 13px，颜色使用次级文字。
- 底部 Tab 三栏：首页、设备、我的；选中态使用主蓝色，非选中态使用次级文字。

## 4. 按钮规则

- 主操作：蓝色实心按钮，白色文字，8px 圆角。
- 次操作：白底蓝色描边，蓝色文字，8px 圆角。
- 列表内快捷操作可使用较小蓝色按钮，但仍保持同一圆角和颜色体系。
- 禁止单页使用大面积黑色按钮、强装饰渐变或与系统不一致的控件样式。

## 4.1 Tab / 分段切换规则

- 用于同一模块内的视图切换，例如 Owner 后台的「用户列表 / 邀请码」。
- Tab 不是提交操作，不使用蓝色实心主按钮样式。
- 同级二级导航使用外层分段 Tab：白底、浅色描边、8px 圆角的容器内放置多个 Tab 项。
- 当前 Tab 使用浅蓝底 `#EAF3FF`、蓝色文字；非当前 Tab 使用白底、次级文字。
- Tab 文案保持 13px 到 14px、600 到 700 字重。
- 页面底部主导航 Tab 仍按三栏底部 Tab 规范执行。

## 5. 页面规则

### 首页

- 必须突出「手上持有：N 台」。
- 「查看列表」是可点击入口，跳转到设备列表的「在我手上」筛选。
- 「扫码借」为蓝色主按钮。
- 「一键还」为白底蓝色描边次按钮。
- 不再展示「待处理」和「申请邀请码」卡片，避免首页信息重复和弱相关。
- 「最新动态」使用白色信息卡片，展示所有设备当前状态，按最新录入或转手时间倒序。

### 设备列表

- 顶部筛选 Tab 仅保留「全部 / 在我手上 / 已借出」。
- 「借入待还」不作为独立 Tab；作为「在我手上」中的列表项状态标签展示。
- 列表项状态标签需区分「在我手上」「借入待还」「已借出」「可借」。
- 非 owner 且非管理员进入借入待还设备详情时，不展示编辑名称和转让 owner 操作。

### 扫码借

- 使用同一页面背景、白色信息卡片和蓝色确认按钮。
- 扫码区域可使用深色模拟相机取景框，但不能影响其他信息卡片风格。

### 管理页

- Owner 后台保持内部工具风格，优先列表、搜索、状态标签。
- 不使用营销式 hero 或大面积装饰元素。
- 用户列表 / 邀请码使用顶部 Tab 切换，Tab 必须可点击，不使用页面底部按钮代替。
- 用户列表页不展示「查看邀请码」按钮。
- 管理员用户列表支持新增、编辑、删除、查询；邀请码页支持无限生成邀请码。

## 6. 验收清单

每次修改原型后检查：

- 页面背景、卡片、按钮、Tab 是否与本规范一致。
- 是否出现单页独有的颜色体系或控件风格。
- 文案是否与 PRD 中业务规则一致。
- 是否有文字溢出、遮挡、重叠或不可读。
- Pencil 是否通过布局问题检查。

前端开发验收时检查：

- 页面是否按原型还原布局、颜色、字号层级、间距、按钮和卡片。
- 页面状态、成功/错误提示、列表项和关键文案是否与原型一致。
- 交互入口和页面跳转是否符合原型表达。
- 若前端实现与原型不一致，必须标记为偏差；除非 PRD 和原型已先更新。
