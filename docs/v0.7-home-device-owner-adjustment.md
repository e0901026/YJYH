# V0.7 首页 / 设备 / Owner 管理调整说明

## 1. 调整目标

本轮根据真机人工验收反馈，收敛首页信息、修正设备筛选口径，并补齐 Owner 管理页的可操作性。

## 2. 首页

- 「查看列表」必须可点击，进入设备列表并默认选中「在我手上」。
- 删除「待处理」卡片。
- 删除「申请邀请码」卡片。
- 「最新动态」改为设备状态流，展示所有当前机器状态。
- 动态按最新时间倒序，最新时间取设备录入时间或最近一次转手 / 借出 / 归还时间。
- 每条动态展示设备名称、当前持有人、状态和最新时间。

## 3. 设备列表

- Tab 从「全部 / 在我手上 / 已借出 / 借入待还」改为「全部 / 在我手上 / 已借出」。
- 「借入待还」不再作为独立 Tab，合并进入「在我手上」。
- 「在我手上」列表包含两类状态：
  - 在我手上：我是 owner 且当前持有人是我。
  - 借入待还：当前持有人是我，但 owner 不是我。
- 借入待还设备点击进入设备详情页。
- 非 owner 且非管理员查看设备详情时，不展示编辑名称和转让 owner 权限。

## 4. Owner / 管理员

- 用户管理页顶部 Tab「用户列表 / 邀请码」必须可点击切换。
- 删除「查看邀请码」按钮。
- 管理员账号可以对用户列表进行新增、编辑、删除、查询。
- 管理员 / Owner 拥有无限邀请码生成能力。

## 5. 原型确认点

需要用户确认以下页面结构后再进入开发：

- 首页：仅保留手上持有、扫码借 / 一键还、最新动态设备状态流。
- 设备列表：三 Tab 结构，借入待还作为在我手上列表项状态。
- 设备详情：权限相关操作按 owner / 管理员显示。
- Owner 管理：顶部 Tab 切换用户列表和邀请码；用户 CRUD 入口与无限邀请码生成入口可见。

## 6. 开发验收点

- 首页「查看列表」跳转可用。
- 首页不再展示待处理和申请邀请码。
- 设备页无「借入待还」Tab。
- 在我手上列表包含「在我手上」和「借入待还」两种状态。
- 借入待还设备详情无编辑名称、转让 owner 操作。
- Owner 用户管理 Tab 可切换。
- Owner 用户列表无「查看邀请码」按钮。
- 管理员用户 CRUD 和无限邀请码生成入口可见。
