# 手机借机管理 App — 项目管理

> 目的：作为本项目的轻量任务控制台，管理需求、原型、开发、测试四个环节的状态，确保每次迭代都有明确入口、范围和验收口径。

## 1. 项目定位

- 项目类型：企业内部移动端工具。
- 平台优先级：Android App 优先开发。
- GitHub 仓库：`https://github.com/e0901026/YJYH`（Public）
- 当前阶段：V0.3 核心流程开发中，第一批 mock 状态闭环已完成。
- 工作方式：单 Agent 负责维护 PRD、原型、开发与测试闭环。
- 最高约束：需求先行，原型确认后再开发，开发必须还原原型。

## 2. 当前版本目标

### V0.1 原型确认版

目标：确认核心业务流程、页面结构和统一视觉风格。

范围：

- 登录 / 注册。
- 首页：手上持有台数、扫码借、一键还、邀请码状态。
- 扫码借：借机人扫码、识别已建档手机、确认借走、通知上一位持有人和设备 owner。
- 手机注册：未建档 IMEI 填写手机名称并建档。
- 一键还：当前待归还列表和单项归还。
- 设备：设备列表、筛选、设备详情、owner 信息。
- 我的：个人信息、邀请码申请、Owner 后台入口。
- Owner 后台：用户列表、邀请码管理。

不在当前版本范围：

- 复杂多 Agent 工作流。
- 后台管理系统独立 Web 端。
- iOS、Web 或多端同步开发。
- 真实后端服务实现和部署。
- 精细权限矩阵之外的高级审批流。
- IMEI 合规导出、审计报表等扩展能力。

## 5. 开发任务拆分

进入开发前必须先完成当前页面的原型确认。

建议开发顺序：

1. Android 项目脚手架与全局样式。
2. Android 页面导航、页面壳、底部导航。
3. 登录与注册。
4. 首页。
5. 扫码借与手机注册流程，优先验证 Android 相机扫码能力。
6. 一键还。
7. 设备列表与设备详情。
8. 我的与邀请码申请。
9. Owner 用户列表与邀请码管理。
10. 状态流、通知模拟、Android 真机测试验收。

每个开发任务必须包含：

- 对应 PRD 章节。
- 对应原型页面。
- 主要交互。
- 完成标准。
- 测试检查点。

## 6. 原型确认规则

页面进入开发前必须满足：

- 页面业务逻辑与 `docs/PRD-手机借机管理.md` 一致。
- 页面视觉与 `docs/system-design-system.md` 一致。
- 页面已在 `prototype/手机借机管理.pen` 中保存。
- 页面已导出到 `prototype/exports` 并能在 `prototype/pencil-prototype.html` 中查看。
- 用户明确表示该页面可进入开发。

如果用户要求调整界面：

1. 先判断是否属于业务需求变更。
2. 若是业务变更，先更新 PRD。
3. 再更新 Pencil 原型。
4. 用户确认原型后，必要时再沉淀设计规范。
5. 最后进入开发或修改代码。

## 7. 测试验收规则

每个功能完成后按三层验收：

| 层级 | 验收对象 | 重点 |
|------|----------|------|
| PRD 验收 | 业务规则 | 角色、权限、流程、异常是否正确 |
| 原型验收 | 界面与交互 | 页面结构、文案、按钮、Tab、状态是否还原 |
| 功能验收 | 实现结果 | 跳转、提交、状态变化、错误提示是否可用 |

测试结果必须记录：

- 通过项。
- 偏差项。
- 偏差来源：PRD / 原型 / 开发 / 测试。
- 需要返工的页面或文件。

## 7.1 阶段门禁规则

每完成一个阶段，必须输出阶段判定：

- 完成项。
- 验证结果。
- 是否通过。
- 阻塞项。
- 下一步动作。
- 是否已提交并推送 GitHub。

阶段状态分为：

| 状态 | 含义 | 下一步 |
|------|------|--------|
| 通过 | 本阶段完成且验证通过 | 更新文档，提交推送，进入下一阶段 |
| 条件通过 | 主要产物完成，但存在环境或外部依赖阻塞 | 记录阻塞，优先推进阻塞项 |
| 未通过 | 产物或验证不满足本阶段要求 | 不进入下一阶段，先修复 |

当前阶段判定：

| 阶段 | 状态 | 说明 | 下一步 |
|------|------|------|--------|
| V0.2 Android 工程壳 | 通过 | Android 页面壳、导航、主题、mock 数据已提交；本地 `assembleDebug`、`testDebugUnitTest`、`lintDebug` 已通过 | 安装 Debug APK 到模拟器或真机，执行 V0.2 人工验收 |
| V0.2 人工验收收口 | 通过 | 模拟器可正常显示；App 可安装启动；登录进入首页；底部导航、设备页 `+`、流程页左侧返回、设备图标调整已验收 | 进入 V0.3 核心流程开发 |
