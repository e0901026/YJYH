# V0.4 Android 测试清单

> 范围：真实 Android 扫码能力与后端联调准备版。进入开发前必须先确认 PRD 和 Pencil 原型。

## 1. 开发准入

- PRD 已写明相机权限、权限拒绝、扫码失败和手动输入 IMEI 兜底规则。
- Pencil 原型已补充并导出 V0.4 新增状态。
- 用户已确认 V0.4 原型可以进入开发。
- Android 技术方案已确认 CameraX + ML Kit Barcode Scanning。

## 2. 自动化检查

- `assembleDebug` 通过。
- `lintDebug` 通过。
- `testDebugUnitTest` 通过。
- 扫码相关解析逻辑覆盖 15 位 IMEI、无效文本、空结果、多码结果。

## 3. 模拟器自测

- 首页点击「扫码借」进入扫码页。
- 相机权限说明态展示正确。
- 权限拒绝态展示正确，并保留手动输入 IMEI 兜底入口。
- 扫码失败 / 无法解析 IMEI 态展示正确。
- 手动输入已建档 IMEI 可进入确认借走。
- 手动输入未建档 IMEI 可进入手机注册。
- 页面截图与 Pencil 原型对照，无明显文字截断、重叠、遮挡。

## 4. 真机自测

- 首次进入扫码页触发系统相机权限申请。
- 允许权限后可进入相机预览。
- 拒绝权限后展示业务提示和重新授权路径。
- 扫描包含 15 位 IMEI 的条码 / 二维码后能解析并进入对应分支。
- 扫描无 IMEI 的条码 / 二维码后展示失败提示。
- 调试日志不输出完整 IMEI。

## 5. 用户人工验收

- 用户确认扫码权限、失败提示和手动兜底路径符合原型。
- 用户确认已建档 / 未建档两条分支仍符合 V0.3 已通过流程。
- 用户确认 V0.4 可进入下一阶段。
