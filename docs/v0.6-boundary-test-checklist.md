## 3. 扫码借边界

| 场景 | 输入 / 操作 | 期望 |
|------|-------------|------|
| 已建档且不在自己手上 | 手动输入 `869301065812347` | 显示小米14 白详情，确认借走后成功或明确失败 |
| 已建档且在自己手上 | 手动输入 `866001123456789` | 显示 OPPO Find X7，确认借走后提示“设备已在当前用户手上” |
| 未建档 IMEI | 手动输入新的 15 位 IMEI | 进入注册手机页 |
| 非法 IMEI | 输入非 15 位数字 | 页面提示“请输入 15 位纯数字 IMEI 编码” |

## 4. 注册手机边界

| 场景 | 输入 / 操作 | 期望 |
|------|-------------|------|
| 新 IMEI 建档 | 未建档 IMEI + 手机名称 | 建档成功，显示成功反馈 |
| 重复建档 | 已存在 IMEI 进入建档接口 | 页面显示“设备已建档”或等价错误 |
| 后端不可用 | 关闭本地后端后提交 | 页面显示建档失败，不假成功 |

## 5. 一键还 / 催还机边界

| 场景 | 输入 / 操作 | 期望 |
|------|-------------|------|
| 借出设备催还 | 我借出去的 tab 点击“催还机” | 显示已催还反馈 |
| 借入设备归还 | 我借入的 tab 点击“一键还” | 显示已归还反馈 |
| 重复点击 | 操作进行中连续点击 | 不重复提交，保留一个结果反馈 |
| 后端不可用 | 关闭本地后端后操作 | 页面显示失败提示 |

## 6. 日志与埋点

执行后检查：

```sh
$ANDROID_HOME/platform-tools/adb logcat -d -t 700 | rg "PhoneLoanAnalytics|AndroidRuntime|FATAL EXCEPTION|Cleartext|NetworkOnMainThread|visible_error|remote_.*failed"
$ANDROID_HOME/platform-tools/adb shell run-as com.yjyh.phoneloan wc -l files/analytics-events.jsonl
```

通过标准：

- 无崩溃。
- 无明文 HTTP 阻断。
- 无主线程网络错误。
- 成功路径埋点队列为 `0`。
- 失败路径至少有可见错误提示和错误埋点。
