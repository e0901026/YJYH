# V0.2 Android 测试清单

> 范围：Android 工程壳、主题、页面路由、底部导航、mock 页面。

## 1. 构建检查

- `./gradlew :app:assembleDebug` 可执行。
- `./gradlew :app:testDebugUnitTest` 可执行。
- `./gradlew :app:lintDebug` 可执行。
- 若本机缺 Android SDK，需要记录为环境问题，不视为产品功能失败。
- 当前 Codex 环境已通过 Android Studio 自带 JBR 和项目 Gradle Wrapper 完成本地构建门禁。
- Debug APK：`android-app/app/build/outputs/apk/debug/app-debug.apk`。

最近验证：2026-05-11

- `:app:assembleDebug`：通过。
- `:app:testDebugUnitTest`：通过，当前无单元测试源码，结果为 `NO-SOURCE`。
- `:app:lintDebug`：通过，剩余依赖升级与应用图标为非阻塞 warning。

人工验收收口：2026-05-11

- 模拟器 `YJYH_API35_UI` 可正常显示，黑屏问题已通过 `-gpu host` 启动参数规避。
- App 可安装并启动到登录页。
- 登录页点击登录可进入首页。
- 首页 / 设备 / 我的底部导航可用于 V0.2 验收。
- 设备列表右上角添加入口为 `+`。
- 流程页返回入口已调整到标题栏左侧。
- 设备图标已调整为手机/设备语义。
- V0.2 人工验收结论：通过，允许进入 V0.3。

## 2. 启动检查

- App 可安装。
- App 可启动到登录页。
- 登录页展示工号、密码、登录、使用邀请码注册。

## 3. 页面导航

- 登录后进入首页。
- 注册页可以从登录页进入。
- 首页底部导航显示：首页、设备、我的。
- 设备页底部导航选中设备。
- 我的页底部导航选中我的。
- 扫码借、手机注册、一键还、设备详情、Owner 页面不显示底部导航。

## 4. 原型还原

- 页面背景为浅灰。
- 卡片为白底浅描边。
- 主按钮为蓝底白字。
- 次按钮为白底蓝描边。
- 设备列表筛选为 Tab，而不是主按钮。
- Owner 用户列表 / 邀请码为页面顶部 Tab。
- 流程页返回入口位于标题栏左侧，不出现在右上角。
- 设备列表标题栏右侧添加入口为 `+` 图标。
- 底部导航的设备图标能明确表达手机/设备。

## 5. V0.2 不验收

- 不验收真实登录。
- 不验收真实扫码。
- 不验收真实后端数据。
- 不验收真实通知推送。
